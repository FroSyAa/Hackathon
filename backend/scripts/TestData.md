# Тестовые данные (детерминированные)

Этот файл описывает формат `TestData.json` — источник детерминированных тестовых данных, которые использует `createTestData.js` для заполнения БД.

Файлы:
- `TestData.json` — основной источник сущностей (пользователи с паролями в открытом виде, названия курсов, соответствия преподавателей).
- `generatedTestData.json` — результат работы `createTestData.js`: реальные ID созданных объектов и пути к файлам в `backend/uploads`.

Коротко о поведении:
- Скрипт `createTestData.js` читает `TestData.json` (если файл есть) и создаёт/обновляет пользователей, профили (teacher/student), направления, курсы, материалы, задания и сдачи.
- Файлы (аватарки, изображения курса, материалы) копируются в `backend/uploads/*` с детерминированными именами. Если исходные файлы отсутствуют в среде запуска, скрипт создаёт плейсхолдеры с детерминированными именами.
- Пароли в `TestData.json` хранятся в открытом виде для удобства тестирования; при сохранении в БД они хэшируются (через `User` модель), но `generatedTestData.json` содержит исходные пароли из `TestData.json` для быстрого логина в тестах.

Редактируйте `TestData.json`, чтобы поменять логины/пароли/имена/курсы. Сохраняйте структуру.

Пример записи пользователя в `TestData.json`:
```json
{
  "email": "teacher1@test.local",
  "password": "passTeacher1",
  "firstName": "Иван",
  "lastName": "Петров",
  "role": "teacher",
  "direction": "Backend",
  "avatarIndex": 0
}
```

Запуск сидера (рекомендуется внутри Docker-контейнера):
```powershell
docker-compose exec backend node scripts/createTestData.js
```
